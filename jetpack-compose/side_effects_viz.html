🚀 LaunchedEffect

Launches a coroutine tied to the composable's lifecycle. Re-runs when the key changes. Auto-cancelled on leave.

@Composable
fun UserProfile(userId: String) {
    var user by remember { mutableStateOf<User?>(null) }

    LaunchedEffect(userId) { // re-runs when userId changes
        user = repo.getUser(userId) // suspend call — non-blocking
    } // auto-cancelled when composable leaves
}

🧹 DisposableEffect

For effects needing cleanup. The onDispose block is always called on leave or key change. Not a coroutine.

@Composable
fun LocationTracker(onLocation: (LatLng) -> Unit) {
    val context = LocalContext.current

    DisposableEffect(context) {
        val mgr = context.getSystemService<LocationManager>()
        val listener = LocationListener { onLocation(it.toLatLng()) }
        mgr.requestLocationUpdates("gps", 0L, 0f, listener) // register

        onDispose {
            mgr.removeUpdates(listener) // cleanup — always runs
        }
    }
}

🔁 SideEffect

Runs after every successful recomposition. Not a coroutine. Used to sync Compose state outward to non-Compose systems.

@Composable
fun AnalyticsScreen(name: String, analytics: Analytics) {
    SideEffect {
        // ✅ Runs after every successful recomposition
        analytics.setScreen(name)
    }
    // Also for system UI:
    val uiCtrl = rememberSystemUiController()
    SideEffect { uiCtrl.setStatusBarColor(Color.Transparent) }
}
