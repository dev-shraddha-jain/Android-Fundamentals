⚡ Smart Recomposition — Only What Changed

When state changes, Compose only recomposes the composables that read that state. Everything else is skipped. This is the key performance advantage over XML.

🛡️ Preventing Unnecessary Recomposition

// ❌ Unstable — recomposes every time parent recomposes
@Composable
fun UserCard(user: User) { // User must be @Stable or data class
    Text(user.name)
}

// ✅ Stable — Compose can SKIP recomposition if params haven't changed
@Stable
data class User(val id: Int, val name: String)

// ✅ remember — cache expensive computation across recompositions
val sortedList = remember(users) { users.sortedBy { it.name } }

// ✅ derivedStateOf — only recompute when derived value actually changes
val isScrolled by remember { derivedStateOf { scrollState.value > 0 } }
