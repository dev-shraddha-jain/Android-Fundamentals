🔷 MVVM — Model · View · ViewModel

The View observes state exposed by the ViewModel. The ViewModel fetches data from the Model (Repository/UseCase). Communication is bidirectional but loose — the ViewModel has no reference to the View.

✅ MVVM Code Pattern

// ViewModel
class UserViewModel : ViewModel() {
    private val _uiState = MutableStateFlow<UiState>(UiState.Loading)
    val uiState = _uiState.asStateFlow()    // View observes this

    fun loadUser(id: String) {
        viewModelScope.launch {
            _uiState.value = UiState.Loading
            _uiState.value = UiState.Success(repo.getUser(id))
        }
    }
}

// View (Compose)
@Composable
fun UserScreen(vm: UserViewModel = hiltViewModel()) {
    val state by vm.uiState.collectAsStateWithLifecycle()
    Button(onClick = { vm.loadUser("42") }) { Text("Load") }
}

🔵 MVI — Model · View · Intent

A strict unidirectional data flow. The View emits Intents (user actions). The ViewModel processes them into a new State. The View renders state. State is immutable — a new copy is always produced.

✅ MVI Code Pattern

// Intent — sealed class of all user actions
sealed class UserIntent {
    data class LoadUser(val id: String) : UserIntent()
    object Refresh : UserIntent()
}

// State — single immutable data class
data class UserState(
    val isLoading: Boolean = false,
    val user: User? = null,
    val error: String? = null
)

// ViewModel processes intents via a Channel
class UserViewModel : ViewModel() {
    private val intentChannel = Channel<UserIntent>(Channel.UNLIMITED)
    private val _state = MutableStateFlow(UserState())
    val state = _state.asStateFlow()

    init {
        viewModelScope.launch {
            intentChannel.consumeAsFlow().collect { intent ->
                reduce(_state.value, intent)
            }
        }
    }

    private suspend fun reduce(current: UserState, intent: UserIntent) {
        when(intent) {
            is UserIntent.LoadUser -> {
                _state.value = current.copy(isLoading = true)
                val user = repo.getUser(intent.id)
                _state.value = current.copy(isLoading = false, user = user)
            }
        }
    }
    fun sendIntent(i: UserIntent) = viewModelScope.launch { intentChannel.send(i) }
}

🎯 When to Choose Which?

Choose MVVM when…
  • Standard CRUD / list screens
  • Small to medium teams
  • Google Architecture Components native
  • Tight deadlines / less boilerplate needed
  • New Android developers on the team
Choose MVI when…
  • Complex multi-state screens (payment, checkout)
  • Need time-travel debugging
  • Many concurrent user actions
  • High test coverage requirement
  • Team experienced with Redux/Elm pattern
